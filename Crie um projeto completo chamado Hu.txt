Crie um projeto completo chamado HubStore com frontend (site responsivo) e backend (serverless functions) pronto para deploy no Vercel. Produza código limpo, moderno e organizado em Português (pt-BR). Entregue tudo em uma estrutura de arquivos que eu possa copiar para um repositório Git.
OBJETIVO DO PROJETO:
Criar uma plataforma chamada HubStore, um catálogo digital moderno (estilo vitrine/loja), onde o usuário visualiza produtos em formato de cards, clica na imagem e pode comprar via PIX ou Cartão usando o gateway AbacatePay. O projeto deve gerar tanto o SITE quanto a base para APP (PWA pronto ou base para integrar ao app nativo).
REQUISITOS DO FRONTEND (HUBSTORE)
Interface estilo vitrine/loja: grade responsiva de produtos em cards (desktop 3-4 colunas, tablet 2, mobile 1).
Cada card deve ter:
imagem grande
título
descrição curta
preço
botão “Comprar”
Ao clicar no card, abrir um modal de produto com:
carrossel de imagens
descrição completa
preço
botão Pagar com PIX
botão Pagar com Cartão
Fluxo PIX:
frontend faz POST para /api/create-payment com { method: "pix", product }
backend retorna QR code base64 e/ou payload PIX
frontend exibe QR em canvas, além de botão “copiar código PIX”
Fluxo Cartão:
frontend faz POST para /api/create-payment com { method: "card", product }
backend retorna checkout_url vindo da AbacatePay
frontend redireciona o usuário
Estilo visual:
moderno, minimalista, com cores: azul (#1E90FF), branco e cinza suave
tipografia limpa
layout fluido e bonito
Criar arquivos:
frontend/index.html
frontend/styles.css
frontend/app.js
Usar window.location.origin como BACKEND_URL automaticamente.
REQUISITOS DO BACKEND (HUBSTORE)
O backend deve ser construído como Vercel Serverless Functions:
Criar:
/api/create-payment.js
/api/webhook.js
Regras:
backend em Node.js 18+
usar axios
ler variáveis de ambiente via process.env
código limpo e organizado
Variáveis de ambiente obrigatórias:
ABACATEPAY_API_URL
ABACATEPAY_BEARER_TOKEN
WEBHOOK_SECRET (opcional)
Fluxo create-payment:
Receber { method, product }
Calcular valor em centavos
Criar cobrança via AbacatePay usando Bearer Token
Para PIX → retornar: qr_base64, payload, expires_at
Para cartão → retornar: checkout_url
Headers: Authorization: Bearer ${ABACATEPAY_BEARER_TOKEN}
Fluxo webhook:
Receber POST da AbacatePay
Responder 200 imediatamente
Incluir comentário sobre como validar assinatura com WEBHOOK_SECRET
